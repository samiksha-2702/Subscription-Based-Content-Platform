Java OOP Concepts

Learn Object-Oriented Programming concepts in Java including classes, inheritance, polymorphism, interfaces, and encapsulation.

Class & Object

A class is a blueprint for creating objects, while an object is an instance of a class. 


class Student{

    String name;
    int marks;

}

Student s = new Student();
  • Class defines properties and methods.
  • Object accesses the class data and behavior.
💡 Objects are created using the new keyword.

Constructor

A constructor initializes objects automatically when they are created. 


class A{

    A(){

        System.out.println("Constructor");

    }

}
  • Constructor name must match the class name.
  • Constructors do not have a return type.
  • Used for object initialization.

Inheritance

Inheritance allows one class to acquire properties and methods of another class. 


class A{

    void show(){}

}

class B extends A{

}
  • extends keyword is used for inheritance.
  • Promotes code reusability.
  • Java supports single, multilevel, and hierarchical inheritance.

Polymorphism

Polymorphism allows methods to perform different behaviors.

Two main types:
  • Compile-Time Polymorphism → Method Overloading
  • Run-Time Polymorphism → Method Overriding

Method Overloading

Method overloading means multiple methods with the same name but different parameters. 


int add(int a, int b){

    return a + b;

}

double add(double a, double b){

    return a + b;

}
  • Achieved at compile time.
  • Improves method flexibility.

Method Overriding

Method overriding allows a subclass to provide its own implementation of a parent method. 


class A{

    void show(){}

}

class B extends A{

    void show(){}

}
  • Occurs in inheritance.
  • Supports runtime polymorphism.

Interface

An interface contains abstract methods that classes must implement. 


interface Shape{

    void draw();

}
  • Used for abstraction.
  • Supports multiple inheritance in Java.
  • Methods are public and abstract by default.

Encapsulation

Encapsulation hides data using private variables and provides controlled access using methods. 


private int data;

public int getData(){

    return data;

}
  • Improves data security.
  • Achieved using getters and setters.

Abstraction

Abstraction hides implementation details and shows only important features. 


abstract class Animal{

    abstract void sound();

}
  • Achieved using abstract classes and interfaces.
  • Helps reduce complexity.

  • Overloading vs overriding
