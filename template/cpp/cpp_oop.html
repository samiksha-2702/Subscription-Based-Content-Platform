C++ Object-Oriented Programming

Core OOP concepts used in interviews and system design

Foundation

Class and Object

A class is a blueprint, and an object is an instance of that class. It allows grouping data and functions together. 

class Student {
public:
    string name;

    void show() {
        cout << name;
    }
};

Student s;
s.name = "Amit";
s.show();
  • Class defines structure
  • Object holds actual data
  • Encapsulates data + behavior
Initialization

Constructor

A constructor is a special function that is automatically called when an object is created. 

class A {
public:
    A() {
        cout << "Object Created";
    }
};
  • Same name as class
  • No return type
  • Used for initialization
Reusability

Inheritance

Inheritance allows one class to acquire properties of another class. It supports code reusability. 

class A {
public:
    void showA() {}
};

class B : public A {
public:
    void showB() {}
};
  • Base class → Parent class
  • Derived class → Child class
  • Supports reuse of code
Flexibility

Polymorphism

Polymorphism means “many forms”. It allows the same function name to behave differently.

Runtime Polymorphism (Function Overriding)

class Base {
public:
    virtual void show() {
        cout << "Base Class";
    }
};

class Derived : public Base {
public:
    void show() override {
        cout << "Derived Class";
    }
};
  • Achieved using virtual functions
  • Supports dynamic behavior
Security

Encapsulation

Encapsulation means wrapping data and methods together and restricting direct access. 

class Bank {
private:
    int balance;

public:
    void setBalance(int b) {
        balance = b;
    }

    int getBalance() {
        return balance;
    }
};
  • Data hiding using private variables
  • Access via public methods
  • Improves security and control
Revision

Key OOP Concepts Summary

  • Class → blueprint of object
  • Object → instance of class
  • Constructor → initializes object
  • Inheritance → reusability of code
  • Polymorphism → multiple forms of behavior
  • Encapsulation → data protection

These concepts are heavily asked in technical interviews and form the foundation of object-oriented design.
